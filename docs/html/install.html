3 Installation

This chapter explains how to install the Jakarta Mail API TCK software.

After installing the software according to the instructions in this chapter, proceed to Chapter 4, "Setup and Configuration," for instructions on configuring your test environment.

3.1 Obtaining a Compatible Implementation

Each compatible implementation (CI) will provide instructions for obtaining their implementation. Jakarta Mail is a compatible implementation which may be obtained from http://eclipse-ee4j.github.io/mail/

3.2 Installing the Software

Before you can run the Jakarta Mail TCK tests, you must install and set up the following software components:

  • Java SE 8

  • Apache Ant 1.10.0+

  • A CI for Jakarta Mail 2.0, one example is Jakarta Mail

  • Jakarta Mail TCK version 2.0

  • The Jakarta Mail 2.0 Vendor Implementation (VI)

Follow these steps:

  1. Install the Java SE 8 software, if it is not already installed.
    Download and install the Java SE 8 software from http://www.oracle.com/technetwork/java/javase/downloads/index.html. Refer to the installation instructions that accompany the software for additional information.

  2. Install the Jakarta Mail TCK 2.0 software.

    1. Copy or download the Jakarta Mail TCK software to your local system.
      You can obtain the Jakarta Mail TCK software from the Jakarta EE site https://jakarta.ee/specifications/mail/.

    2. Use the unzip command to extract the bundle in the directory of your choice:
      unzip jakarta-mail-tck-2.0.0.zip
      This creates the TCK directory. The TCK is the test suite home, <TS_HOME>.

  3. Install the Apache Ant 1.10.0+ software, if it is not already installed.
    Download and install Apache Ant 1.10.0+ software from Apache Ant Project. For complete information about Ant, refer to the extensive documentation on the Apache Ant Project site. The Apache Ant Manual is available at http://ant.apache.org/manual/index.html. Apache Ant is protected under the Apache Software, License 2.0, which is is available on the Apache Ant Project license page at http://ant.apache.org/license.html.

  4. Install a Jakarta Mail 2.0 Compatible Implementation.
    A Compatible Implementation is used to validate your initial configuration and setup of the Jakarta Mail TCK 2.0 tests, which are explained further in Chapter 4, "Setup and Configuration."
    The Compatible Implementations for Jakarta Mail are listed on the Jakarta EE Specifications web site: https://jakarta.ee/specifications/mail/.

  5. Install the Jakarta Mail VI to be tested.
    Follow the installation instructions for the particular VI under test.
